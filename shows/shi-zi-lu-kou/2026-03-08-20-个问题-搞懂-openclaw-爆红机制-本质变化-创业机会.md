然后，2B 的三个机会，我想讲。第一个，是短期内最清晰的一个商业路径。其实和 2C 一样，就它是套客。就是把复杂的 Open Cloud 的技术封装成面向特定的企业的这样的工种的医用的产品。不管是给 sales，给律师的，给财务的等等等等。就这些垂直的场景，谁能够先把开箱己用接入了 IM 里的这个 Agent 的一个数字员工，数字同事给做出来，谁就能够最先的拿到第一批的客户。我觉得这个是非常值得的，这个低垂果实。

然后，第二个呢，在 2B 领域，有一个非常关键的事情，就是安全和隐私。我们都在这一期播客里面聊了很多了，就是企业对安全和隐私的在乎。那是远远大于 2C 的。那这里面，也有一些新的公司在动。比如说，这个 RunLayer，它拿了 1100 万美元的投资，它在做企业把 OpenCloud 控制的这个一个平台。另外一个叫 Citorra，我看到他做的也挺有趣的。他是在做每一个 Agent 启动的时候，要去验证你的身份。就防止有被恶意的 Agent 来进入我们的企业这种生产环境的可能性。这里面，挺百花齐放的。我也不是专家，但是我感觉，如果做不好的话，所有的垂直的应用都是空中楼阁。所以，这里应该会有非常多的机会。

然后，最后，我想讲的是，护身盒的问题。其实，Agent 在企业里面，每跑一次任务，它都会积累数据。就是哪一步 Agent 出错了，人类又是怎么去纠正它的。那最后，它是用什么样的路径做对的。就所谓的这个执行的轨迹，就叫 Trajectory。它是不可能在公开的训练数据集里面有的。但是呢，有了这样的执行轨迹，它是非常能显著的提升 Agent 在一个特定的企业环境下的准确率。所以，一个 AI Agent，如果他能够先进到一个企业里面去，他先积累足够多的执行的轨跡。其实，这也是一种意义的锁定用户，形成里的这个护城河的一个机会。所以，可能今天还是要先更快的动起来，进到企业的生态里面。其实，本身快就是一个护城河。

我们的第 19 个问题是，OpenCloud 的这一波热潮，它会是一个短期的现象吗？我感觉，这个又是一个短期的，又是一个长期的东西。这是什么意思呢？就是，从目前来看，OpenCloud 这种现象级的产品，它的潜力或者说它的能力，大概率是高估的。它未来还是得收敛到它实际产品能力能支撑的市场上去。我对它的目前的感觉是，这个产品仍然非常的粗糙，有很多槽点和可以改进的地方。所以，我觉得短期来看，它的能力是被高估的，会回归。但是，从长期来看，它展示了一个非常重要的潮流。或者说，它背后的这些关键要素，实际上都已经全了。比如说，机模的能力已经在那里了。或者说，这种数据飞轮，如刚才不论是 2B 说这种过程的飞轮过程的积累，还是 2C 说这种 Skill 的积累，也在这里了。它也跑通了很多原型的商业场景，包括像豪大刚才说的，代替了很多 app 作为通用入口。这些都是现实可行的。但是，它又需要很多很多的 polish。比如说，刚才说的 token 太多，太贵的问题。它的记忆还是比较粗糙。很多时候，不 work。GOR 也可以再打磨等等。所以，我觉得，从这个角度来看，我们可能站在一个百花齐放的时代的起点。未来会有很多的公司去积极探索这个模块，我怎么打磨那个模块，怎么打磨。二者之间，可以怎么组合。最后，可能还是会收敛。但是，未必会收敛到 OpenCloud 上去。还是会收敛到一个远远更加精细，甚至我们现在都没有办法想象到的一种产品形态上去。

我们今天的第二十个问题，最后一个问题是，我们如果 12 个月后回看 OpenCloud，觉得最值得被记住的会是什么？我觉得，最值得记住的，有一个现在大家已经发现的，然后最值得被记住的，有一个我觉得大家没发现，但是我希望大家未来记住的东西。一个是，大家已经发现的是，Agentic AI 很牛逼。然后，我觉得大家会记住，工具很多会构成复利，然后积累的经验会构成复利，Feedback loop，特别重要。这些跟 Agentic AI 相关，大家会记住，哇，这个很厉害。如果我要做 AI，我一定要把它做上 Agentic 的。但是我觉得，大家没有意识到的一点是，AI 的能力是一方面。但是，它可能只是一个放大器。人或者说，用户的能力，可能更关键。比如，两个一样的 Open Cloud 像我们用的，可能和另外一个，对 AI 没有什么经验的普通人，哪怕这个普通人是很厉害的程序员，用起来会很不一样。可能我们的效果，就会比他的要好很多。未来，随着 AI 越来越成，熟能力越来越强。这样的杠杆效应，可能会更严重。所以，我的一个，不知道是暴论还是，我希望大家认识到的一点是，人会不会用 AI，在未来可能是一个更大的一个区分因素。它可能，反而是 OpenCloud 会给我们带来的一个启发。即使，最终 OpenCloud 是一个短周期现象。但是，它也带来了很多的一些启发。它证明了一种，以小博大，以及，以大众用户为主的这样的一个 Agent 的设计哲学。所谓，以小见大，就是说，它通过几个 lockdown 文件，以 soul.mdmemory.md，还有 heartbeats.md 等等这些，还有 identity.md 这些。让 Agent 具备了一个人格。他有自己的记忆，还有心跳机制。不需要那么复杂的系统，也不需要大量的这种工程。简单却有效，且直给的方式。让大家意识到，原来这是我的助手，他和我一起成长。我觉得，这是非常有启发的。
